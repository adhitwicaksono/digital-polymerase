# Prototype 001B: RNA → HNA Template-Guided Reconstruction

**Project:** Digital Polymerase  
**Broader context:** XNA World Project  
**Prototype:** 001B  
**Conversion:** RNA → HNA  
**Method:** Full-template-guided HNA scaffold reconstruction  
**Current status:** Successful 8-mer candidate generator; not physically validated  
**Documentation status:** Preliminary; to be extended after long-input / 111-mer HNA scaling is complete

---

## Summary

Prototype 001B is the refactored RNA → HNA converter in Digital Polymerase.

The original HNA converter was the first RNA → XNA success in the project. Prototype 001B revisits that early result with better structure, clearer CLI usage, and stronger validation reporting.

Unlike ANA 002A.2 and FANA 003A, which use chain-preserving local reconstruction, HNA 001B uses a **full HNA template as the scaffold donor**. This is because HNA is structurally more divergent from RNA and has a hexitol-like scaffold.

Current validated use case:

```text
RNA 8-mer + HNA 8-mer template → HNA-like 8-mer candidate
```

## Current Method

For each residue:

1. select the corresponding HNA template residue by position
2. extract HNA backbone/scaffold atoms from the template
3. align RNA local sugar atoms to the HNA template frame
4. transform RNA nucleobase atoms into the HNA frame
5. combine HNA scaffold atoms with transformed RNA base atoms
6. export an HNA-like residue
7. validate chain continuity and base-attachment distances

This is a full-template method. It is not designed to extend an 8-mer template to a 12-mer or larger RNA input.

## Current Boundary

Prototype 001B requires:

```text
HNA template length >= source RNA length
```

This means an 8-mer HNA template can convert an 8-mer RNA input, but not a 12-mer RNA input.

A failed 12-mer attempt is expected and scientifically correct:

```text
RNA-12mer.pdb + 481d-HNA8nt.pdb → rejected
```

because the converter refuses to silently recycle a shorter HNA template.

---

## Why HNA Needs a Separate Scaling Prototype

HNA is more structurally divergent than ANA and FANA.

For HNA scaling, we likely need:

```text
fragment-guided reconstruction
segment-guided reconstruction
overlapping-window reconstruction
chain-continuity enforcement
base-attachment validation
```

This should become:

```text
Prototype 001C
```

not an expansion of 001B by silent template cycling.

## Known Limitations

- 8-mer full-template case only.
- Not a long-RNA converter.
- Not energy-minimized.
- No force-field parameters.
- No topology files.
- No explicit `CONECT` records.
- HNA ring/scaffold stereochemistry requires expert review.
- Biological function is not implied.
