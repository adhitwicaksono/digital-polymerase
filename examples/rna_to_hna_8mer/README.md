# Example: RNA 8-mer → HNA Candidate Structure

This example demonstrates **Prototype 001** of Digital Polymerase: a template-guided reconstruction workflow for converting a short RNA oligomer into an HNA-like candidate structure.

The goal is to test whether an RNA input structure can be transformed into an HNA candidate model by using an experimental HNA structure as a backbone template.

---

## Overview

This example uses:

1. an input RNA 8-mer PDB structure
2. an HNA 8-mer template PDB structure
3. the Prototype 001 RNA → HNA converter

The converter does **not** simply insert atoms into RNA. Instead, it performs a template-guided reconstruction:

1. parse the RNA input PDB
2. parse the HNA template PDB
3. extract the HNA backbone geometry
4. preserve the RNA sequence and nucleobase atoms
5. align RNA bases onto the HNA residue geometry
6. transplant transformed bases onto the HNA backbone
7. export a candidate HNA-like PDB structure

## Running the Example

From the repository root:

```bash
python src/digital_polymerase/prototypes/rna_to_hna_template_based.py \
  --rna examples/rna_to_hna_8mer/input_rna_8mer.pdb \
  --template examples/rna_to_hna_8mer/hna_8mer_template.pdb \
  --output examples/rna_to_hna_8mer/converted_hna_candidate.pdb \
  --base-policy preserve \
  --report examples/rna_to_hna_8mer/conversion_report.md
```

---

## Base Policy

The recommended default is:

```bash
--base-policy preserve
```

This preserves RNA base chemistry:

```text
A → 6HA
U → 6HU
G → 6HG
C → 6HC
```

This avoids implying that uracil has been chemically converted into thymine.

A label-level option is also available:

```bash
--base-policy rna-to-dna-label
```

This maps:

```text
U → 6HT
```

However, this should be interpreted cautiously. This option changes the residue label but does not automatically add the thymine methyl group. Therefore, it should not be treated as a true chemical uracil-to-thymine conversion.

## Expected Result

The output PDB should contain an HNA-like candidate structure generated by combining:

- HNA backbone geometry from the template
- transformed RNA nucleobase atoms from the source structure
- residue names assigned according to the selected base policy

The generated structure should include `REMARK` lines stating that the model is computationally generated and requires further validation.

---

## Important Scientific Note

This example produces a **candidate structure**, not a validated HNA molecule.

The output is not:

- energy-minimized
- molecular-dynamics validated
- force-field parameterized
- experimentally verified
- proof of chemical or biological function

Before using the output for scientific interpretation, the structure should be inspected and refined.

Recommended downstream checks include:

1. visual inspection in PyMOL, ChimeraX, VMD, or similar software
2. residue completeness checks
3. bond-length and bond-angle inspection
4. clash detection
5. comparison with the original HNA template
6. energy minimization
7. molecular dynamics simulation, if appropriate
8. expert review of the HNA backbone chemistry

---

## What This Example Demonstrates

This example demonstrates that RNA → HNA conversion is more realistically approached through **template-guided reconstruction** than through direct atom insertion.

Earlier naive approaches can preserve RNA-like sugar geometry or distort the backbone. By using an HNA template, the converter can better approximate HNA backbone geometry while preserving source sequence information.

This supports the broader Digital Polymerase principle:

> Compare source and target structures first, infer the transformation logic, then reconstruct candidate models using polymer-aware rules.

---

## Current Limitations

This example is intentionally small and experimental.

Known limitations:

- designed for short oligomers
- tested primarily on an 8-mer case
- depends strongly on template quality
- does not generate force-field parameters
- does not guarantee chemically valid stereochemistry
- does not write full molecular connectivity for all viewers
- may require manual inspection of inferred bonds in molecular visualization tools

This example should be treated as a development case for building more general NA → XNA reconstruction workflows.
