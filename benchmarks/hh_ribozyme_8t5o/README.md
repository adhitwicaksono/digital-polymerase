# Benchmark 002: HH Ribozyme RNA → HNA Failure Benchmark

**Project:** Digital Polymerase  
**Broader context:** XNA World Project  
**Benchmark:** 002  
**Conversion attempted:** RNA → HNA  
**Status:** Archived failure / stress-test benchmark  
**Recommended interpretation:** Productive failure; not a validated HNA conversion

---

## Summary

Benchmark 002 documents an early attempt to scale the RNA → HNA conversion logic from a short 8-mer case to a larger folded RNA structure: an HH-type I ribozyme.

The benchmark was created after **Prototype 001: RNA → HNA 8-mer template-guided reconstruction** showed that a short RNA input could be converted into an HNA-like candidate structure using an experimental HNA template.

When the same general idea was applied to a larger folded ribozyme, the method did not generalize cleanly. The result revealed that large structured RNAs require a different conversion strategy from short oligomers.

This benchmark is kept to document the failure mode and guide future development.

## Background

Prototype 001 showed that RNA → HNA candidate generation could work for a short 8-mer when a comparable HNA template was available.

The successful 8-mer strategy was:

```text
RNA 8-mer input
+
experimental HNA 8-mer template
+
local Kabsch alignment / base transplantation
=
candidate HNA-like PDB
```

Benchmark 002 asked whether this logic could scale to a larger folded RNA structure.

The answer was: **not directly**.

---

## What This Benchmark Tests

This benchmark tests whether RNA → HNA conversion can preserve:

- larger RNA architecture
- base positions and stacking
- global ribozyme fold
- local sugar/backbone geometry
- inter-residue backbone continuity
- plausible HNA-like local chemistry

It also tests whether a short-template or local-edit approach can be extended to a large folded RNA.

---

## Main Result

The benchmark showed that the early conversion methods were not sufficient for a large folded ribozyme.

The best interpretation of the structure-preserving output is:

> a fold-preserving pseudo-HNA local perturbation model

It should **not** be interpreted as:

> a successfully converted or chemically validated HNA ribozyme

---

## Why It Failed

The HH ribozyme is not just a longer 8-mer.

It is a folded RNA structure with:

- long-range tertiary geometry
- base stacking
- possible catalytic-site constraints
- inter-residue backbone continuity
- local and global conformational dependencies

Changing the sugar/backbone chemistry affects more than individual residues. It can affect the backbone path, base orientation, torsions, chain continuity, and tertiary contacts.

A method that works for a short oligomer does not automatically work for a structured ribozyme.

## Lessons for Digital Polymerase

Benchmark 002 established an important rule:

> Short-mer success does not imply folded-RNA success.

It also introduced a distinction between two development modes:

### Mode 1: Short-Oligomer Template Reconstruction

For controlled small structures such as:

```text
RNA 8-mer → HNA candidate
short RNA duplex → XNA candidate
short single-stranded RNA → XNA candidate
```

### Mode 2: Large Folded-RNA Stress Testing

For larger structures such as:

```text
hammerhead ribozyme
aptamers
riboswitches
structured RNA domains
functional RNA motifs
```

Mode 2 requires more constraints, more validation, and likely a different reconstruction strategy.

---

## Recommended Next Strategy

Future large RNA → XNA conversion should not rely on simple local editing alone.

Possible strategies include:

1. **Segment-wise reconstruction**  
   Split the RNA into shorter fragments and reconstruct each segment using compatible XNA templates.

2. **Fragment-chain reconstruction**  
   Use connected XNA fragments and stitch overlapping windows while enforcing chain continuity.

3. **Base-frame preservation**  
   Preserve nucleobase planes and key base-pairing geometry, then rebuild the XNA backbone around the base framework.

4. **Constraint-aware optimization**  
   Preserve important tertiary contacts, catalytic-site geometry, and backbone continuity while minimizing strain.

5. **Energy minimization and MD validation**  
   Treat any converted structure as a candidate that requires refinement and simulation.
